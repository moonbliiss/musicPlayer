?? Схема проекта после добавления сервера
Ваш проект будет выглядеть так:

text
my-music-player/
+-- server/                      # Серверная часть
¦   +-- package.json             # Зависимости сервера (express, cors, music-metadata)
¦   +-- server.js                # Основной файл сервера
¦   L-- .env.example             # (опционально) Пример переменных окружения
+-- src/                         # Фронтенд (React + Vite)
¦   +-- components/              # Ваши компоненты
¦   +-- App.tsx
¦   +-- main.tsx
¦   L-- ...
+-- index.html
+-- vite.config.ts
+-- package.json                 # Зависимости фронтенда (уже есть)
+-- package-lock.json
L-- node_modules/                # Уже есть все нужные пакеты
Сервер будет располагаться в отдельной папке, чтобы не смешивать его с фронтендом. У него будет свой package.json и свои зависимости (которые вы уже скопировали в общую node_modules? Но лучше создать отдельный node_modules для сервера, чтобы избежать конфликтов версий. Однако если вы уже скопировали все зависимости в общую папку, можете использовать их и для сервера, просто указав путь к ним. Но проще сделать отдельный package.json в папке server и установить зависимости там (вы уже их скопировали в общую node_modules, поэтому можно просто создать package.json с нужными зависимостями и при запуске сервера они будут найдены в родительской node_modules, если настроить NODE_PATH. Но для простоты мы создадим отдельную папку server и установим зависимости через npm (но у вас нет интернета). Так как у вас уже есть все пакеты в общей node_modules, вы можете использовать их для сервера, прописав в package.json сервера зависимости, и при запуске Node.js будет искать их в родительской папке? Не совсем. Лучше разместить сервер в корне и использовать общий node_modules. Тогда создадим server.js прямо в корневой папке проекта. Упростим: добавим серверный файл в корень и расширим текущий package.json.

Проще: добавьте файл server.js в корень проекта, и в package.json добавьте зависимости express, cors, music-metadata (они уже есть в node_modules). Тогда запуск сервера: node server.js. А фронтенд запускается отдельно.

Таким образом, структура будет:

text
my-music-player/
+-- server.js                   # Сервер
+-- src/                        # Фронтенд
+-- package.json                # Общий (дописать зависимости для сервера)
+-- node_modules/               # Все зависимости уже здесь
L-- ...
Это проще всего. Давайте так и сделаем.

??? Шаг 1. Добавляем серверный скрипт server.js
Создайте в корневой папке проекта файл server.js со следующим содержимым:

javascript
const express = require('express');
const fs = require('fs').promises;
const path = require('path');
const cors = require('cors');
const musicMetadata = require('music-metadata');

const app = express();
const PORT = 3001;

// ===== НАСТРОЙКА ПУТИ К МУЗЫКЕ =====
// ?? Измените этот путь на вашу папку с музыкой
const MUSIC_DIR = '/home/user/Music'; // ЗАМЕНИТЕ НА СВОЙ ПУТЬ!

// Разрешаем запросы с фронтенда (который обычно на порту 5173)
app.use(cors());

// Раздаём аудиофайлы по URL /audio/...
app.use('/audio', express.static(MUSIC_DIR));

// Эндпоинт: получить список песен с метаданными
app.get('/api/songs', async (req, res) => {
  try {
    // Читаем содержимое папки
    const files = await fs.readdir(MUSIC_DIR);
    // Фильтруем аудиофайлы (можно расширить список)
    const audioExtensions = ['.mp3', '.flac', '.wav', '.ogg', '.m4a', '.aac'];
    const audioFiles = files.filter(file =>
      audioExtensions.includes(path.extname(file).toLowerCase())
    );

    const songs = [];
    for (const fileName of audioFiles) {
      const filePath = path.join(MUSIC_DIR, fileName);
      try {
        // Пытаемся прочитать метаданные
        const metadata = await musicMetadata.parseFile(filePath);
        const { common, format } = metadata;
        songs.push({
          id: fileName,
          title: common.title || path.basename(fileName, path.extname(fileName)),
          artist: common.artist || 'Неизвестный исполнитель',
          album: common.album || 'Неизвестный альбом',
          duration: format.duration || 0,
          url: `/audio/${encodeURIComponent(fileName)}`, // URL для воспроизведения
        });
      } catch (err) {
        // Если не удалось прочитать метаданные, добавляем без них
        console.warn(`Не удалось прочитать метаданные для ${fileName}:`, err.message);
        songs.push({
          id: fileName,
          title: path.basename(fileName, path.extname(fileName)),
          artist: 'Неизвестно',
          album: 'Неизвестно',
          duration: 0,
          url: `/audio/${encodeURIComponent(fileName)}`,
        });
      }
    }
    res.json(songs);
  } catch (err) {
    console.error('Ошибка чтения папки:', err);
    res.status(500).json({ error: 'Не удалось прочитать папку с музыкой' });
  }
});

// Запускаем сервер
app.listen(PORT, () => {
  console.log(`Сервер запущен на http://localhost:${PORT}`);
  console.log(`Читает музыку из: ${MUSIC_DIR}`);
});
Что здесь происходит:

MUSIC_DIR — переменная с путём к папке, откуда берётся музыка. Вы должны изменить её на свой путь (например, '/media/user/Music' или '/home/username/Music').

app.use('/audio', express.static(...)) — отдаёт файлы по URL вида /audio/песня.mp3.

/api/songs — эндпоинт, который сканирует папку, для каждого аудиофайла читает метаданные через music-metadata.parseFile и возвращает массив объектов с нужными полями.

cors() — разрешает запросы с вашего фронтенда (порт 5173 по умолчанию для Vite).

Вариант А: Загружать песни при старте приложения (самый простой)

В вашем главном компоненте (например, App.tsx) добавьте useEffect, который при монтировании делает запрос:

tsx
import React, { useState, useEffect } from 'react';

interface Song {
  id: string;
  title: string;
  artist: string;
  album: string;
  duration: number;
  url: string;
}

function App() {
  const [songs, setSongs] = useState<Song[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState('');

  useEffect(() => {
    const fetchSongs = async () => {
      try {
        const response = await fetch('http://localhost:3001/api/songs');
        if (!response.ok) throw new Error('Ошибка загрузки');
        const data = await response.json();
        setSongs(data);
      } catch (err) {
        setError('Не удалось загрузить песни. Убедитесь, что сервер запущен.');
        console.error(err);
      } finally {
        setLoading(false);
      }
    };
    fetchSongs();
  }, []);

  if (loading) return <div>Загрузка музыки...</div>;
  if (error) return <div style={{ color: 'red' }}>{error}</div>;

  return (
    <div>
      <h1>Мой плеер</h1>
      <ul>
        {songs.map(song => (
          <li key={song.id}>
            <strong>{song.title}</strong> — {song.artist}
            <br />
            <audio controls src={`http://localhost:3001${song.url}`} />
          </li>
        ))}
      </ul>
    </div>
  );
}

export default App;

?? Шаг 5. Настройка прокси в Vite (для удобства)
Чтобы не писать http://localhost:3001 в каждом запросе и избежать CORS-проблем (хотя мы уже включили cors), добавьте прокси в vite.config.ts:

ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': 'http://localhost:3001',
      '/audio': 'http://localhost:3001',
    },
  },
});
Теперь во фронтенде можно писать просто:

ts
fetch('/api/songs');
// и
<audio src={song.url} /> // где song.url начинается с /audio/...
И не нужно указывать полный URL с портом.

?? Шаг 6. Запуск всего проекта
Теперь нужно запускать два процесса:

Сервер: в одном терминале выполните node server.js.

Фронтенд: в другом терминале (или после запуска сервера) выполните npm run dev.

Они будут работать независимо. Vite-сервер по умолчанию на порту 5173, а сервер на 3001. Прокси перенаправит запросы с /api и /audio на сервер.

?? Дополнительно: как сделать поле для ввода пути (опционально)
Если вы хотите, чтобы пользователь мог ввести путь вручную, можно добавить:

На сервере новый эндпоинт для обновления пути:

javascript
app.use(express.json()); // добавляем парсинг JSON

let currentMusicDir = MUSIC_DIR; // изначальный путь

app.post('/api/set-path', (req, res) => {
  const { path: newPath } = req.body;
  if (!newPath) return res.status(400).json({ error: 'Путь не указан' });
  // Проверяем, существует ли папка
  try {
    const stats = fs.statSync(newPath);
    if (!stats.isDirectory()) {
      return res.status(400).json({ error: 'Указанный путь не является папкой' });
    }
    currentMusicDir = newPath;
    // Обновляем статический путь (можно пересоздать middleware, но проще перезапустить сервер)
    // В данном случае мы просто меняем переменную, но статика останется старой.
    // Можно перемонтировать express.static, но это сложнее.
    // Проще предложить перезапустить сервер после смены пути.
    res.json({ success: true, newPath });
  } catch (err) {
    res.status(500).json({ error: 'Ошибка доступа к папке' });
  }
});
Но тогда нужно обновить и express.static — это не делается на лету. Проще перезапускать сервер с новым путём. Поэтому для простоты задайте путь в коде один раз.

?? Итог
Серверная часть на Node.js + Express читает файлы из заданной папки и отдаёт их через HTTP.

Фронтенд больше не использует webkitdirectory или showDirectoryPicker, а просто запрашивает список песен с сервера при загрузке.

Все диалоги выбора папки исключены, проблема QtWebEngine решена.

Сервер и фронтенд запускаются отдельно, но могут быть объединены в один процесс (например, через concurrently).

Теперь у вас есть полностью локальный плеер, который работает в QtWebEngine без каких-либо проблем с выбором папки.
